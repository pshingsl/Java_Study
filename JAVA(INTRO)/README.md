## 1. 프로그래밍 언어의 이해

컴퓨터는 기본적으로 `0`과 `1`로 이루어진 <strong>기계어(이진 코드)</strong>만 이해할 수 있다. <br>
사람이 컴퓨터와 소통하기 위해 사용하는 중간 도구가 바로 **프로그래밍 언어**

### 🔍 언어의 분류
* **저급 언어 (Low-level Language)**
    * 기계어에 가까운 언어 (예: 어셈블리어)
    * 실행 속도는 매우 빠르지만, 사람이 배우고 유지보수하기가 매우 어렵다.
* **고급 언어 (High-level Language)**
    * 사람이 이해하기 쉬운 단어와 문장으로 구성된 언어 (예: **Java**, Python, C++)
    * 우리가 공부하는 대부분의 언어가 여기에 속하며, 생산성이 높다.

### ⚙️ 컴파일(Compile)이란?
사람이 작성한 **고급 언어** 소스 코드를 컴퓨터가 이해할 수 있는 **기계어**로 번역하는 과정

**[컴파일 및 실행 프로세스]**
> **소스 코드 작성(.java)** ➔ **컴파일러 번역(javac)** ➔ **실행 파일 생성(.class)** ➔ **컴퓨터 실행**


1.소스코드를(.java) 작성한다. <br>
2.컴파일러는 자바소스 코드를 이용하여 클래스 파일을 생성. 
컴파일된 클래스 파일은 JVM이 인식할 수 있는 바이트코드이다.<br>
3.JVM은 클래스 파일의 바이트 코드를 해석하여 바이너리 코드로 변환하고 프로그램을 수행한다. <br> 
4.수행 결과가 컴퓨터터에 반영된다. <br>

## 2. 자바 실행 환경의 핵심 (JDK, JRE, JVM)



### 1️⃣ JVM (Java Virtual Machine)
* **역할:** 자바 프로그램이 실제로 돌아가는 **가상의 컴퓨터**
* **특징:** 운영체제(Windows, Mac, Linux)에 상관없이 자바가 실행될 수 있게 해준다. -> 이로 인해 OS에 종속 받지 않음
* **철학:** *Write Once, Run Anywhere* (한 번 작성하면 어디서든 실행한다!)![img_1.png](img_1.png)

### 2️⃣ JRE (Java Runtime Environment)
* **구성:** `JVM` + `실행에 필요한 라이브러리 파일`
* **역할:** 자바 프로그램을 **실행**만 하고 싶을 때 필요한 환경입니다. (일반 사용자용)

### 3️⃣ JDK (Java Development Kit)
* **구성:** `JRE` + `개발 도구(컴파일러 javac 등)`
* **역할:** 자바 프로그램을 **개발(코드 작성)** 하기 위해 반드시 필요한 도구 모음. 
![img.png](img.png)
### 4️⃣ SDK (Software Development Kit)
* 소프트웨어를 개발하기 위한 도구 모음을 통칭하는 넓은 개념.
* **JDK는 자바 전용 SDK이다**.

## 💡 한 줄 요약 (비유)

| 구분 | 비유 | 설명 |
| :--- | :--- | :--- |
| **JVM** | **주방 기구** | 음식을 실제로 조리(실행)하는 도구 |
| **JRE** | **서빙 접시** | 음식을 먹기(실행) 위해 필요한 최소한의 세팅 |
| **JDK** | **주방 전체** | 음식을 만들기(개발) 위해 필요한 모든 도구와 재료 |
