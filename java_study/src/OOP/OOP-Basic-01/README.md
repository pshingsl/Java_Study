---

## ✅ 1. 객체 지향 프로그래밍이란?

객체 지향 프로그래밍(OOP)은 현실 세계의 사물을 **객체 단위로 나누어 프로그램을 만드는 방식**이다.

👉 쉽게 말하면

> **데이터 + 기능을 하나로 묶어서 관리하는 프로그래밍 방식**

---

## ✅ 2. 객체(OOP.ObjectEx)란?


> 현실 세계에 존재하거나 개념적으로 존재하는 것 중 **다른 것과 구별할 수 있는 대상**



### 객체의 구성 요소

객체는 2가지로 이루어진다.

| 구성 | 의미 |
|------|------|
| 속성 (Field) | 객체의 데이터 (상태) |
| 동작 (Method) | 객체가 수행하는 기능 |



### ✔ 예시

자동차 객체

- 속성 → 색상, 속도, 바퀴 수
- 동작 → 출발, 정지, 가속

---

## ✅ 3. 객체 모델링

현실의 객체를 프로그램으로 표현하는 과정을
**객체 모델링** 이라고 한다.

👉 예

현실 자동차 → 프로그램 속 Car 클래스

---

# ⭐ 4. 객체 지향 프로그래밍의 핵심 특징 (4대 특징)

## 1️⃣ 캡슐화 (Encapsulation)

> **데이터와 기능을 하나로 묶고, 외부로부터 내부 구현을 숨기는 것**

즉, 객체 내부의 중요한 정보는 직접 접근하지 못하게 하고  
정해진 방법(메서드)을 통해서만 접근하도록 만드는 것이다.

---

### ✔ 쉽게 이해하기

ATM 기계 내부 구조는 볼 수 없지만  
버튼을 눌러 돈을 찾을 수 있다.

👉 내부 구조 숨김 = 캡슐화

---

### ✔ 구현 방법

- 필드를 `private`으로 선언
- Getter / Setter 메서드 제공

---

### ✔ 장점

- 데이터 보호
- 잘못된 값 방지
- 코드 안정성 증가
- 유지보수 쉬움

---

## 2️⃣ 상속 (Inheritance)

> **부모 클래스의 필드와 메서드를 자식 클래스가 물려받는 것**

이미 만들어진 클래스를 재사용하여  
새로운 클래스를 쉽게 만들 수 있는 기능이다.

---

### ✔ 쉽게 이해하기

동물 → 부모 클래스  
강아지 → 자식 클래스

강아지는 부모의 기능을 그대로 사용 가능

- 먹는다
- 숨쉰다

---

### ✔ 특징

- 코드 재사용 가능
- 계층 구조 형성
- 유지보수 용이

---

### ✔ 장점

- 중복 코드 제거
- 개발 속도 향상
- 구조 파악 쉬움

---

## 3️⃣ 다형성 (Polymorphism)

> **같은 방식으로 사용하지만 실행 결과가 다양하게 나타나는 성질**

즉, 동일한 메서드를 호출해도  
객체에 따라 다른 동작을 수행한다.

---

### ✔ 쉽게 이해하기

리모컨의 "전원 버튼"

- TV → TV 켜짐
- 에어컨 → 에어컨 켜짐

👉 같은 명령 → 다른 결과

---

### ✔ 구현 방법

- 메서드 오버라이딩
- 자동 타입 변환
- 인터페이스 활용

---

### ✔ 장점

- 코드 유연성 증가
- 확장성 향상
- 유지보수 쉬움

---

## 4️⃣ 추상화 (Abstraction)

> **불필요한 정보는 숨기고 핵심 기능만 표현하는 것**

객체의 복잡한 내부 구현은 감추고  
사용자가 필요한 기능만 제공하는 개념이다.

---

### ✔ 쉽게 이해하기

자동차 운전

- 내부 엔진 구조는 몰라도 됨
- 핸들, 페달만 알면 운전 가능

👉 필요한 기능만 노출 = 추상화

---

### ✔ 구현 방법

- 추상 클래스 (abstract class)
- 인터페이스 (interface)

---

### ✔ 장점

- 복잡성 감소
- 코드 이해 쉬움
- 설계 구조 명확

---

# ⭐ 한눈에 보는 4대 특징 정리

| 특징 | 핵심 의미 |
|------|------------|
| 캡슐화 | 내부 숨기고 보호 |
| 상속 | 부모 기능 재사용 |
| 다형성 | 같은 호출, 다른 결과 |
| 추상화 | 핵심만 표현 |

---

# ⭐ 핵심 한 줄 정리

> 객체 지향 프로그래밍의 4대 특징은  
> **캡슐화 · 상속 · 다형성 · 추상화** 이다.

> **현실 세계처럼 객체 중심으로 프로그램을 만드는 방식이다.**
