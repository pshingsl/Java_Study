## 멀티 스레드 (Multi Thread)

운영체제는 실행 중인 프로그램을 **프로세스(Process)** 로 관리한다.

---

## 멀티 태스킹 (Multi Tasking)

멀티 태스킹은 **두 가지 이상의 작업을 동시에 처리하는 것**을 의미한다.

👉 반드시 멀티 프로세스를 의미하지는 않는다.

---

## 예시

- 메신저 프로그램
    - 채팅 수행
    - 파일 전송 수행

👉 하나의 프로세스에서 여러 작업 수행 → **멀티 스레드**

---

## 스레드(Thread)

스레드는 **코드의 실행 흐름**을 의미한다.

- 스레드 1개 → 하나의 실행 흐름
- 스레드 2개 → 두 개의 실행 흐름

---

## 핵심 개념

- 멀티 프로세스 → 프로그램 단위 병렬 처리
- 멀티 스레드 → 하나의 프로그램 내부 병렬 처리

---

# 멀티 프로세스 vs 멀티 스레드

| 구분 | 멀티 프로세스 | 멀티 스레드 |
|------|--------------|-------------|
| 실행 단위 | 프로세스 | 스레드 |
| 메모리 | 독립적 | 공유 |
| 안정성 | 높음 | 낮음 |
| 속도 | 상대적으로 느림 | 빠름 |
| 영향 범위 | 서로 영향 없음 | 하나 오류 시 전체 영향 |

---

## 핵심 차이 설명

### 멀티 프로세스
- 서로 독립적인 메모리 사용
- 하나의 프로세스 오류 → 다른 프로세스 영향 없음

---

### 멀티 스레드
- 같은 프로세스 내에서 실행
- 메모리 공유 (Heap 영역)
- 하나의 스레드 예외 발생 → 전체 프로세스 종료 가능

---

# 실행 흐름 구조

## 멀티 프로세스

```
프로세스 A (메모리 독립)
프로세스 B (메모리 독립)
```

---

## 멀티 스레드

```
프로세스 (공유 메모리)
 ├─ 스레드 1
 ├─ 스레드 2
 └─ 스레드 3
```

---

# 실무 예시

## 1. 메신저

- 채팅 수신 스레드
- 파일 전송 스레드
- UI 처리 스레드

---

## 2. 서버 프로그램

```text
클라이언트 요청 1 → 스레드 1 처리
클라이언트 요청 2 → 스레드 2 처리
클라이언트 요청 3 → 스레드 3 처리
```

👉 동시에 여러 요청 처리 가능

---

## 3. 병렬 데이터 처리

```java
// 리스트 데이터를 병렬 처리
list.parallelStream()
    .forEach(data -> process(data));
```

👉 CPU 코어를 활용한 병렬 처리

---

## 4. 안드로이드 (또는 UI 프로그램)

- 메인 스레드 → UI 처리
- 백그라운드 스레드 → 네트워크 통신

---

# 멀티 스레드 사용하는 이유

- 성능 향상 (병렬 처리)
- 사용자 경험 개선 (UI 멈춤 방지)
- 서버 처리량 증가

---

# 주의 사항

## 1. 동기화 문제 (Race Condition)

```java
count++; // 여러 스레드가 동시에 접근 시 문제 발생
```

👉 해결: synchronized, Lock 사용

---

## 2. 데드락 (Deadlock)

```text
스레드 A → 자원1 점유 → 자원2 대기
스레드 B → 자원2 점유 → 자원1 대기
```

👉 서로 무한 대기 상태

---

# 핵심 정리

- 스레드 = 실행 흐름
- 멀티 스레드 = 하나의 프로세스 내부 병렬 처리
- 메모리 공유 → 빠르지만 위험
- 서버, UI, 병렬 처리에서 필수 개념
- 동기화 문제 반드시 고려해야 함
